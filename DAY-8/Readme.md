# Day 8 – Object-Oriented Programming (OOP) in Python

**Date:** 4 July 2026

## Overview

The eighth day of the Agentic AI Training introduced the fundamentals of **Object-Oriented Programming (OOP)** in Python. The session focused on understanding the core concepts of classes and objects, their importance in software development, and how they help model real-world entities in code. We also explored constructors, reference variables, and their role in object creation. To reinforce these concepts, we implemented a restaurant management structure using OOP in class, and I independently designed an OLA/Uber model using the same principles.

## 1. Introduction to Object-Oriented Programming

* Learned the concept of **Object-Oriented Programming (OOP)** and its importance in modern software development.
* Understood how OOP helps organize programs into reusable, modular, and maintainable components.
* Discussed the advantages of OOP, including code reusability, scalability, and better organization of large applications.

---

## 2. Classes and Objects

* Understood that a **Class** acts as a blueprint or template for creating objects.
* Learned that an **Object** is an instance of a class containing its own data and behavior.
* Explored how classes define the properties (attributes) and actions (methods) of objects.
* Studied several examples to understand the relationship between classes and objects.

---

## 3. Real-World and Computer Science Applications

* Explored how real-world entities can be represented as objects in programming.
* Learned to identify classes and objects from everyday scenarios.
* Discussed examples such as students, vehicles, restaurants, and banking systems.
* Understood how OOP is used to design scalable software systems by modelling real-world problems.

---

## 4. Constructors in Python

* Learned the purpose of constructors in object creation.
* Understood the working of the `__init__()` method.
* Studied how constructors initialize object attributes automatically when an object is created.
* Explored how constructors simplify object initialization and improve code readability.

---

## 5. Reference Variables

* Learned the concept of **reference variables** in Python.
* Understood that a reference variable stores the memory address of an object rather than the object itself.
* Discussed how multiple reference variables can point to the same object in memory.
* Explored the role of reference variables while accessing object attributes and methods.
